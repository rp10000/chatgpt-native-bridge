# 常见问题

## 这是 OpenAI API wrapper 吗？

不是。它不调用 OpenAI API，也不需要 API key。

## 它会抓取 ChatGPT 网页吗？

不会。它不读取 DOM、cookies、localStorage、IndexedDB、token 或网络请求。

## 为什么不自动控制浏览器？

浏览器自动化脆弱，也容易碰到不该碰的边界。这个项目保留用户对 ChatGPT 网页会话的可见控制。

## 可以选择自动或手动吗？

可以，但这里的自动是“自动准备”，不是“自动发送”：

```bash
cgn open latest --mode manual
cgn open latest --mode assist
cgn open latest --mode auto
```

`manual` 只打印路径。`assist` 会打开 ChatGPT 并复制 `01_PASTE_TO_CHATGPT.md`。`auto` 还会打开 outbox 文件夹，方便你看到并选择附件。它不会自动粘贴、上传、点击发送，也不会抓取 ChatGPT 输出。

## 我需要 ChatGPT Plus 或 Pro 吗？

bridge 本身不要求。ChatGPT 网页端哪些功能可用，取决于你的账号和工作区。

## 可以用 ChatGPT Projects 吗？

可以，而且推荐用。把 `.chatgpt-native/project-instructions.md` 粘到 Project instructions。

## 可以上传图片吗？

可以。用：

```bash
cgn ask --task "Review UI" --type ux-review --include-screenshots "screenshots/*.png"
```

然后在 ChatGPT 里上传 outbox 中的截图。

## 什么东西不要上传？

不要上传 `.env`、私钥、cookies、session dump、Authorization header、token、密码或任何你不愿意交给 ChatGPT 的内容。

## Codex 怎么知道什么时候用 bridge？

`cgn init` 会安装 `.agents/skills/chatgpt-native-bridge/SKILL.md`。你也可以明确告诉 Codex：

```text
Use chatgpt-native-bridge for this task.
```
