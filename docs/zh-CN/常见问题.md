# 常见问题

## 这是 OpenAI API wrapper 吗？

不是。它不调用 OpenAI API，也不需要 API key。

## 它会抓取 ChatGPT 网页吗？

不会。它不读取 DOM、cookies、localStorage、IndexedDB、token 或网页请求。

## 现在主路径是什么？

`v0.6.0` 开始，主路径是桌面客户端：

```bash
npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn start
```

客户端负责 Pro 剪贴板接力、Thinking MCP 启动和写回 Codex 提示。

## GPT-5.5 Pro 为什么不用 MCP？

当前设计按产品边界处理：Pro 用客户端剪贴板接力，Thinking 可走 MCP 工具。这样不用 API，也不需要浏览器插件或网页抓取。

## MCP 会给 ChatGPT shell 权限吗？

不会。MCP 只暴露有限工具：读取项目状态、安全读取文件和 diff、生成 handoff、写回 inbox。不提供任意 shell、任意写文件、commit 或 push。

## 为什么不能自动创建 ChatGPT app？

自动创建 ChatGPT app 需要浏览器自动化或未公开网页接口。这个项目不做这类操作。客户端会尽量把要复制的 URL 和下一步说明准备好。

## 可以手动使用吗？

可以：

```bash
cgn handoff --task "复核这个项目"
cgn done
```

## 可以上传图片吗？

可以。用 handoff 时带截图：

```bash
cgn handoff --task "Review UI" --type ux-review --include-screenshots "screenshots/*.png"
```

## 什么内容不要交给 ChatGPT？

不要上传 `.env`、私钥、cookies、session dump、Authorization header、token、密码或任何你不愿交给 ChatGPT 的内容。
