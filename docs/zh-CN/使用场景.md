# 使用场景

## 推荐使用

MCP 可用时，优先启动：

```bash
cgn mcp serve --host 127.0.0.1 --port 47832
cgn mcp config
```

让 ChatGPT 通过 MCP 工具读取受限上下文、diff 或 handoff 文件。下面的命令是 MCP 不可用时的 Markdown fallback 示例。

| 场景 | 命令示例 |
| --- | --- |
| 复杂需求拆解 | `cgn handoff --task "..." --type plan,requirements` |
| 架构重构前复核 | `cgn handoff --task "..." --type architecture --include-files "src/**/*.js"` |
| 页面设计/文案批判 | `cgn handoff --task "..." --type ux-review,naming-copy --include-screenshots "screenshots/*.png"` |
| 需要网页搜索或深度研究 | `cgn handoff --task "..." --type research` |
| 图片方向或视觉提示词 | `cgn handoff --task "..." --type image-direction` |
| Codex 完成后的二次复核 | `cgn handoff --task "..." --type diff-review --include-diff --include-tests` |

## 不推荐使用

不要为了这些任务使用 bridge：

- 只改错别字
- 只改格式
- 明确的测试失败修复
- lockfile-only 更新
- 包含不适合上传到 ChatGPT 的秘密信息

## 判断标准

如果任务只需要本地确定性执行，让 Codex 直接做。

如果任务需要产品判断、研究、截图分析、视觉方向或第二视角复核，用 bridge。
