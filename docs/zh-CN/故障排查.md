# 故障排查

## `npx` 找不到

先安装 Node.js 和 npm。项目要求 Node 18 或更新版本。

## npm 包还没发布

发布到 npm registry 前，请用：

```bash
npx github:rp10000/chatgpt-native-bridge init
```

或本地安装：

```bash
git clone https://github.com/rp10000/chatgpt-native-bridge.git
cd chatgpt-native-bridge
npm link
cgn --help
```

## `cgn open` 没打开浏览器

先 dry-run：

```bash
cgn open latest --dry-run
```

然后手动打开 `https://chatgpt.com`，复制 outbox 里的 `ask.md`。

如果想只看路径：

```bash
cgn open latest --mode manual
```

如果想同时打开 ChatGPT 和 outbox 文件夹：

```bash
cgn open latest --mode auto
```

## 剪贴板复制失败

打开 `cgn open` 输出的 outbox 路径，手动复制 `ask.md`。

## 看不到应该上传什么

运行：

```bash
cgn open latest --mode manual
```

看输出里的：

```text
Paste prompt
Upload/select in ChatGPT
Outbox
```

## 找不到 latest handoff

先看状态：

```bash
cgn status
```

如果没有 handoff，先创建：

```bash
cgn ask --task "Review onboarding UX" --type plan,ux-review
```

## ChatGPT 回复没有导入

复制 ChatGPT 回复后运行：

```bash
cgn done
```

或保存成文件：

```bash
cgn import latest ./reply.md
```

## Codex 没有使用 Skill

明确告诉 Codex：

```text
Use chatgpt-native-bridge for this task.
```

并检查：

```bash
cgn doctor
```

## secret guard 拦截了文件

说明文件路径或内容看起来像敏感信息。请删除敏感内容、写安全摘要，或不要把该文件放进 handoff。
