# 故障排查

## `npx` 找不到

先安装 Node.js 和 npm。项目要求 Node 18 或更新版本。

## npm 包还没发布

发布到 npm registry 前，请用：

```bash
npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn setup --mcp
```

或本地安装：

```bash
git clone https://github.com/rp10000/chatgpt-native-bridge.git
cd chatgpt-native-bridge
npm link
cgn --help
```

## Codex 里没有 MCP server

先安装 Codex MCP 配置：

```bash
npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn setup --mcp
```

然后重启 Codex，或打开一个新的 Codex 线程。

如果你使用手动 HTTP server 备用路径，再启动本地 server：

```bash
cgn mcp serve --host 127.0.0.1 --port 47832
```

然后打开：

```text
http://127.0.0.1:47832/health
```

应该看到一个简短 JSON health 响应。

## MCP 端口被占用

换一个端口：

```bash
cgn mcp serve --host 127.0.0.1 --port 47833
cgn mcp config --port 47833
```

ChatGPT 端要使用对应的 `/mcp` endpoint。

## ChatGPT 访问不了 `127.0.0.1`

有些 ChatGPT MCP 连接方式不能直接访问你的本机 loopback 地址。请使用账号或工作区支持的官方 MCP 连接方式，例如 Secure MCP Tunnel。

不要用抓取 ChatGPT 网页、提取 cookies、读取 localStorage 或隐藏接口来解决。

## ChatGPT 说没有 bridge 工具

如果 ChatGPT 说不能调用 `review_current_project`、`write_to_codex` 或 `api_tool`，说明应用可能被选中了，但工具没有暴露给当前对话。

先检查：

```bash
cgn mcp wait --timeout 30
cgn mcp trace
```

如果没有观察到工具调用：

1. 先看 `cgn mcp trace` 打印的 `Latest ChatGPT Server URL`。
2. 保持 tunnel 命令运行。Cloudflare quick tunnel URL 是临时的。
3. 在 ChatGPT app 设置里确认它使用的就是这个最新 `https://.../mcp` Server URL。
4. 刷新工具。如果仍然看不到工具，删除 app，用最新 URL 重新创建。
5. Authentication 选择 `No authentication`。
6. 打开新的 ChatGPT 对话，选择 Developer mode，勾选 `chatgpt-native-bridge`，然后让它使用 `write_to_codex` 写回 Codex。

请使用 `0.4.1` 或更新版本。旧版本缺少完整的 ChatGPT app 工具描述、顶层 `securitySchemes`、请求 trace、session `GET /mcp` SSE 支持、GPT Actions 写回备用路径、本地 MCP agent 工具，以及 npx 命令分隔符修复。

如果 request log 里能看到 `openai-mcp` 的 `initialize` 和 `tools/list`，但 tool-call log 里没有新的 `tools/call`，说明服务端已经能被 ChatGPT 访问并扫描。剩下的问题在 ChatGPT 端：当前聊天没有暴露这个 app、当前模式不支持 custom app、app metadata 过期，或者模型没有选择调用工具。

如果这里仍然卡住，先使用 Markdown 备用路径：

```bash
cgn handoff --task "Review this project" --type diff-review
cgn done
```

或者使用 GPT Actions 写回备用路径：

```text
cgn mcp connect 打印：
  https://example.trycloudflare.com/mcp

把这个 OpenAPI URL 导入 Custom GPT Action：
  https://example.trycloudflare.com/action/openapi.json

如果 ChatGPT 显示 `Something went wrong`，就不要继续点 URL 导入，直接把这个文件里的 schema JSON 粘贴进去：
  .chatgpt-native/actions/openapi.json

然后对这个 Custom GPT 说：
  先调用 review_current_project。
  只在需要时读取相关文件。
  最后调用 write_to_codex，把最终 Markdown 建议写回 Codex。
```

这条路使用 ChatGPT 官方 GPT Actions/OpenAPI，不依赖 MCP write action。它只会写入 `.chatgpt-native/inbox`。

## 我以为 MCP 会有 shell 工具

不会。这个项目让 ChatGPT 通过受限 MCP 工具提供建议，再由 Codex 在本地执行。

查看当前 MCP 工具：

```bash
cgn mcp config
```

## `cgn open` 没打开浏览器

先 dry-run：

```bash
cgn open latest --dry-run
```

然后手动打开 `https://chatgpt.com`，复制 outbox 里的 `01_PASTE_TO_CHATGPT.md`。

如果想只看路径：

```bash
cgn open latest --mode manual
```

如果想同时打开 ChatGPT 和 outbox 文件夹：

```bash
cgn open latest --mode auto
```

## 剪贴板复制失败

打开 `cgn open` 输出的 outbox 路径，手动复制 `01_PASTE_TO_CHATGPT.md`。

## 看不到应该上传什么

运行：

```bash
cgn open latest --mode manual
```

看输出里的：

```text
Paste prompt
Upload/select in ChatGPT
Outbox
```

## 找不到 latest handoff

先看状态：

```bash
cgn status
```

如果没有 handoff，先创建：

```bash
cgn handoff --task "Review onboarding UX" --type plan,ux-review
```

## ChatGPT 回复没有导入

复制 ChatGPT 回复后运行：

```bash
cgn done
```

或保存成文件：

```bash
cgn import latest ./reply.md
```

## Codex 没有使用 Skill

明确告诉 Codex：

```text
Use chatgpt-native-bridge for this task.
```

并检查：

```bash
npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn doctor
```

## secret guard 拦截了文件

说明文件路径或内容看起来像敏感信息。请删除敏感内容、写安全摘要，或不要把该文件放进 handoff。
