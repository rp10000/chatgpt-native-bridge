# 故障排查

## 客户端打不开

先确认命令：

```bash
cgn start --dry-run
cgn desktop --dry-run
```

如果没有安装桌面客户端，命令会打开 GitHub Release 页面。开发环境可以运行：

```bash
npm install
npm run desktop:dev
```

## ChatGPT 没有调用工具

先运行：

```bash
cgn mcp trace
```

如果没有请求，说明 ChatGPT 没有访问当前 Server URL。重新运行：

```bash
cgn mcp connect --yes --open
```

然后在 ChatGPT 里刷新或重建 connector。

## ChatGPT 访问错项目

ChatGPT 默认只能访问客户端当前选中的项目，不能看全局文件系统。

如果 ChatGPT 提示“不是当前连接项目”：

1. 回到桌面客户端。
2. 点击 `选择项目`，选中真正要操作的项目。
3. 点击 `连接 ChatGPT`。
4. 回到 ChatGPT，刷新工具或重建 connector。

不要把其它项目路径直接交给 ChatGPT。需要换项目时，先在客户端切换。

## URL 无效

ChatGPT 不能直接使用 `localhost`。需要 `https://.../mcp` URL。使用：

```bash
cgn mcp connect --yes --open
```

命令会启动临时 tunnel 并复制 Server URL。

## 交接报告没有生成

正常主流程是让 ChatGPT 在网页端调用 `create_handoff_report`。

如果没有报告：

1. 在 ChatGPT 里直接说：`请生成交接报告。`
2. 回到客户端查看状态灯是否变成紫色。
3. 如果仍然没有，运行：

```bash
cgn mcp trace
```

确认 ChatGPT 是否真的调用了工具。

## Pro 回复没有自动导入

检查：

1. 是否先打开了 `Pro 辅助规划`。
2. 是否复制的是 Pro 的完整回复。
3. 回复里是否保留了客户端要求的开头和结尾标记。

如果自动识别失败，在客户端诊断区手动粘贴回复并导入。Pro 只是备用规划方式，不能直接读取本地项目。

## 什么时候用手动备用

如果桌面客户端和 MCP 都不可用：

```bash
cgn handoff --task "复核这个项目"
cgn done
```
