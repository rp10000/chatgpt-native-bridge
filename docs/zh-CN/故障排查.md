# 故障排查

## 客户端打不开

先确认命令：

```bash
cgn start --dry-run
cgn desktop --dry-run
```

如果没有安装桌面客户端，命令会打开 GitHub Release 页面。开发环境可以运行：

```bash
npm install
npm run desktop:dev
```

## Pro 回复没有自动写回

检查：

1. 是否先点击了 `Pro 深度规划`。
2. 是否复制的是 Pro 的完整回复。
3. 回复里是否保留了客户端要求的开头和结尾标记。

如果自动识别失败，在客户端的“手动写回 / 诊断”里粘贴回复并导入。

## ChatGPT 看不到 MCP 工具

运行：

```bash
cgn mcp trace
```

如果没有请求，说明 ChatGPT 没有访问当前 Server URL。重新运行：

```bash
cgn mcp connect --yes --open
```

然后在 ChatGPT 里刷新或重建 connector。

## URL 无效

ChatGPT 不能直接使用 `localhost`。需要 `https://.../mcp` URL。用：

```bash
cgn mcp connect --yes --open
```

命令会启动临时 tunnel 并复制 Server URL。

## 什么时候用手动 fallback

如果桌面客户端和 MCP 都不可用：

```bash
cgn handoff --task "复核这个项目"
cgn done
```
