# 快速开始

普通用户从桌面客户端开始。客户端会隐藏 MCP、隧道和路径细节。

## 1. 打开客户端

在你要处理的项目目录里运行：

```bash
npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn start
```

客户端主界面只有三个主按钮：

- `连接 ChatGPT`
- `开始复核`
- `交给 Codex`

## 2. 主路径：Thinking / MCP

1. 点 `连接 ChatGPT`。
2. 在 ChatGPT 里创建或刷新这个工具。
3. 点 `开始复核`。
4. 把复制好的这句话发给 ChatGPT：

```text
请使用 chatgpt-native-bridge 复核当前项目，并把最终建议写回 Codex。
```

5. 等 ChatGPT 调用工具并写回。
6. 点 `交给 Codex`，把复制好的句子发给 Codex。

给 Codex 的句子是：

```text
读取最新 Bridge 回复，然后继续执行。
```

## 3. Pro 辅助规划

只有在你想让 GPT-5.5 Pro 根据打包上下文做辅助规划时，才使用 `Pro 辅助规划`。

边界很重要：

```text
Pro 不能直接读取本地项目。
Pro 只能看客户端复制过去的上下文。
```

流程：

1. 展开 `Pro 辅助规划`。
2. 点 `复制 Pro 上下文`。
3. 粘贴给 GPT-5.5 Pro。
4. 复制 Pro 的回复。
5. 客户端会把匹配的回复导入 Codex inbox。
6. 点 `交给 Codex`。

## 4. 第一次安装到 Codex

在当前项目里安装 Skill 和 MCP 配置：

```bash
npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn setup --mcp
```

如果 Codex 提示需要重启，就重启 Codex，或者开一个新线程。

## 5. 手动备用方式

如果桌面客户端或 ChatGPT 工具不可用：

```bash
cgn handoff --task "复核这个项目"
cgn done
```

Codex 最后读取：

```text
.chatgpt-native/inbox/{id}/reply.md
.chatgpt-native/inbox/{id}/CODEX_READ_THIS.md
```
