# 快速开始

普通用户从桌面客户端开始。客户端会隐藏 MCP、隧道和路径细节。

## 1. 打开客户端

在你要处理的项目目录里运行：

```bash
npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn start
```

客户端流程：

```text
选择项目 -> 连接 ChatGPT -> 开始处理 -> 查看结果 -> 交给 Codex
```

## 2. 主路径：Thinking / MCP

1. 点 `连接 ChatGPT`。
2. 在 ChatGPT 里创建或刷新这个工具。
3. 点 `开始处理`。
4. 把复制好的句子发给 ChatGPT。
5. 等 ChatGPT 调用工具并写回。
6. 点 `查看结果`，检查命令输出、文件变更和最新回复。
7. 点 `交给 Codex`，把复制好的句子发给 Codex。

给 Codex 的句子是：

```text
读取最新 Bridge 回复，检查变更摘要，然后继续执行、测试和总结。
```

## 3. Pro 辅助规划

只有在你想让 GPT-5.5 Pro 根据打包上下文做辅助规划时，才使用 Pro 辅助规划。

边界：

```text
Pro 不能直接读取本地项目。
Pro 只能看客户端复制过去的上下文。
```

## 4. 项目白名单

当前项目可以直接打开。如果想让 ChatGPT 切换到其他项目，先添加：

```bash
cgn projects add D:\path\to\project
```

## 5. 第一次安装到 Codex

在当前项目里安装 Skill 和 MCP 配置：

```bash
npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn setup --mcp
```

如果 Codex 提示需要重启，就重启 Codex，或者开一个新线程。

## 6. 手动备用方式

如果桌面客户端或 ChatGPT 工具不可用：

```bash
cgn handoff --task "复核这个项目"
cgn done
```
