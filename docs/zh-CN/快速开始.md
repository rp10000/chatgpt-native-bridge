# 快速开始

普通用户从桌面客户端开始。客户端会隐藏 MCP、隧道和路径细节。

## 1. 打开客户端

在你要处理的项目目录里运行：

```bash
npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn start
```

客户端流程：

```text
选择项目 -> 连接 ChatGPT -> 在 ChatGPT 网页端处理 -> 生成交接报告
```

## 2. 主路径：Thinking / MCP

1. 点 `连接 ChatGPT`。
2. 在 ChatGPT 里创建或刷新这个工具。
3. 在 ChatGPT 里发送：`请使用 chatgpt-native-bridge 打开当前连接项目。`
4. 继续描述你要 ChatGPT 处理的任务。
5. 让 ChatGPT 读取、修改、运行必要检查，并调用 `show_changes`。
6. 让 ChatGPT 调用 `create_handoff_report`，或在客户端点 `生成交接报告`。
7. 让 Codex 复核交接报告和真实 diff。

给 Codex 的句子是：

```text
读取最新 Bridge 交接报告，检查真实 diff，然后复核、测试、提交和总结。
```

## 3. Pro 辅助规划

只有在你想让 GPT-5.5 Pro 根据打包上下文做辅助规划时，才使用 Pro 辅助规划。

边界：

```text
Pro 不能直接读取本地项目。
Pro 只能看客户端复制过去的上下文。
```

## 4. 项目边界

ChatGPT 默认只能操作客户端当前选中的项目。要处理其它项目，先在客户端切换项目，再重新连接。

## 5. 第一次安装到 Codex

在当前项目里安装 Skill 和 MCP 配置：

```bash
npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn setup --mcp
```

如果 Codex 提示需要重启，就重启 Codex，或者开一个新线程。

## 6. 手动备用方式

如果桌面客户端或 ChatGPT 工具不可用：

```bash
cgn handoff --task "复核这个项目"
cgn done
```
