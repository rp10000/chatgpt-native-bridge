# 快速开始

这个页面面向第一次使用 `chatgpt-native-bridge` 的用户。

## 1. 初始化

npm 发布前：

```bash
npx github:rp10000/chatgpt-native-bridge init
```

npm 发布后：

```bash
npx chatgpt-native-bridge init
```

本地开发：

```bash
git clone https://github.com/rp10000/chatgpt-native-bridge.git
cd chatgpt-native-bridge
npm link
cgn init
```

初始化会生成：

```text
.agents/skills/chatgpt-native-bridge/SKILL.md
.chatgpt-native/project-instructions.md
.chatgpt-native/outbox/
.chatgpt-native/inbox/
```

## 2. 设置 ChatGPT Project

打开 ChatGPT，创建 Project：

```text
Codex Native Advisor
```

把下面这个文件内容复制进 Project instructions：

```text
.chatgpt-native/project-instructions.md
```

## 3. 让 Codex 使用 bridge

复制给 Codex：

```text
这个任务如果需要规划、架构批判、UI/UX 复核、命名文案、研究、图片方向或 diff review，
请使用 chatgpt-native-bridge。

你来运行 cgn ask 生成 handoff。
然后运行 cgn open latest。
告诉我需要在 ChatGPT 里粘贴什么、上传什么。
等我运行 cgn done 导入回复后，
你读取 .chatgpt-native/inbox/{id}/reply.md，
只采纳合理建议，继续本地修改、测试和总结。
```

也可以运行：

```bash
cgn guide codex --lang zh-CN
```

## 4. 在 ChatGPT 里操作

`cgn open latest` 会打开 ChatGPT、复制 `01_PASTE_TO_CHATGPT.md`，并在终端输出：

```text
Paste prompt file: 要粘贴的 01_PASTE_TO_CHATGPT.md 路径
Upload/select in ChatGPT: 需要选择或上传的附件
Outbox: 本次 handoff 文件夹
```

你需要：

```text
1. 粘贴 01_PASTE_TO_CHATGPT.md
2. 打开 02_UPLOAD_THESE_FILES.md
3. 按任务需要上传 context.md / diff.patch / screenshots / files
4. 让 ChatGPT 输出 Codex next actions
5. 复制 ChatGPT 最终回复
```

模式选择：

```bash
cgn open latest --mode assist
cgn open latest --mode manual
cgn open latest --mode auto
```

`manual` 只打印路径。`assist` 是默认模式，会打开 ChatGPT 并复制 prompt。`auto` 还会打开 outbox 文件夹，但不会自动粘贴、上传或发送。

## 5. 导回本地

```bash
cgn done
```

然后 Codex 读取：

```text
.chatgpt-native/inbox/{id}/reply.md
```
