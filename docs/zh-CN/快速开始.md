# 快速开始

这个页面面向第一次使用 `chatgpt-native-bridge` 的用户。

## 1. 初始化

npm 发布前：

```bash
npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn setup --mcp
```

npm 发布后：

```bash
npx chatgpt-native-bridge setup --mcp
```

本地开发：

```bash
git clone https://github.com/rp10000/chatgpt-native-bridge.git
cd chatgpt-native-bridge
npm link
cgn setup --mcp
```

初始化会生成项目文件，并把 MCP server 写入 Codex 配置：

```text
.agents/skills/chatgpt-native-bridge/SKILL.md
.chatgpt-native/project-instructions.md
.chatgpt-native/outbox/
.chatgpt-native/inbox/
```

如果没有全局安装 `cgn`，后续命令都可以写成 `npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn <command>`。

## 2. 设置 ChatGPT Project

打开 ChatGPT，创建 Project：

```text
Codex Native Advisor
```

把下面这个文件内容复制进 Project instructions：

```text
.chatgpt-native/project-instructions.md
```

## 3. 让 Codex 使用 bridge

复制给 Codex：

```text
这个任务如果需要规划、架构批判、UI/UX 复核、命名文案、研究、图片方向或 diff review，
请使用 chatgpt-native-bridge。

先检查 .chatgpt-native/inbox 是否已有新回复，有就读取 reply.md 和 CODEX_READ_THIS.md。
优先走 MCP 主路径。如果 ChatGPT 还没连接，请运行或提示我运行 npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn mcp connect --yes --open。
如果 MCP 还没安装，请运行或提示我运行 npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn setup --mcp，然后重启 Codex 或打开新线程。
如果 MCP 不可用，再运行 cgn handoff 生成并打开 handoff，告诉我需要在 ChatGPT 里粘贴什么、上传什么。
等 ChatGPT 自动写回，或我运行 cgn done 导入回复后，
你读取 .chatgpt-native/inbox/{id}/reply.md 和 CODEX_READ_THIS.md，
只采纳合理建议，继续本地修改、测试和总结。
```

也可以运行：

```bash
cgn guide codex --lang zh-CN
```

## 4. MCP 主路径

首次安装会写入 Codex MCP 配置。如果项目已经初始化，但 Codex 里还没有 MCP，运行：

```bash
cgn mcp install
```

ChatGPT 网页连接：

```bash
cgn mcp connect --yes --open
```

按命令打印的字段创建 ChatGPT connector。然后在 ChatGPT 里直接说：

```text
请使用 chatgpt-native-bridge 复核这个项目。
自动检查当前项目状态和 diff，需要上下文就读取相关文件，
最后把你的建议写回本地 Codex。
```

ChatGPT 应该自动把建议写回本地 inbox。

手动 HTTP server 备用路径：

```bash
cgn mcp serve --host 127.0.0.1 --port 47832
```

## 5. 备用：在 ChatGPT 里手动操作

`cgn handoff` 会打开 ChatGPT、复制 `01_PASTE_TO_CHATGPT.md`，并在终端输出：

```text
Paste prompt file: 要粘贴的 01_PASTE_TO_CHATGPT.md 路径
Upload/select in ChatGPT: 需要选择或上传的附件
Outbox: 本次 handoff 文件夹
```

你需要：

```text
1. 粘贴 01_PASTE_TO_CHATGPT.md
2. 打开 02_UPLOAD_THESE_FILES.md
3. 按任务需要上传 context.md / diff.patch / screenshots / files
4. 让 ChatGPT 输出 Codex next actions
5. 复制 ChatGPT 最终回复
```

模式选择：

```bash
cgn open latest --mode assist
cgn open latest --mode manual
cgn open latest --mode auto
```

新手优先用 `cgn handoff --mode ...`。`cgn open latest --mode ...` 是高级拆分流程里的单独打开步骤。

`manual` 只打印路径。`assist` 是默认模式，会打开 ChatGPT 并复制 prompt。`auto` 还会打开 outbox 文件夹，但不会自动粘贴、上传或发送。

## 6. 导回本地

```bash
cgn done
```

然后 Codex 读取：

```text
.chatgpt-native/inbox/{id}/reply.md
```
