# 快速开始

这个页面面向第一次使用 `chatgpt-native-bridge` 的用户。

## 1. 打开桌面客户端

在你要处理的项目目录里运行：

```bash
npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn start
```

也可以用：

```bash
cgn desktop
cgn client
```

客户端只有三个主按钮：

- `Pro 深度规划`
- `Thinking 工具复核`
- `写回 Codex`

## 2. Pro 深度规划

适合 GPT-5.5 Pro 不能直接调用 Apps/MCP 的情况。

```text
1. 点 Pro 深度规划。
2. 客户端复制提示词并打开 ChatGPT。
3. 把提示词粘贴给 Pro。
4. 复制 Pro 的回复。
5. 客户端自动写回 Codex inbox。
6. 点 写回 Codex，把复制好的句子发给 Codex。
```

## 3. Thinking 工具复核

适合当前 ChatGPT 模式可以调用 MCP 工具的情况。

```text
1. 点 Thinking 工具复核。
2. 客户端启动本地 MCP server 和 tunnel。
3. 在 ChatGPT 里创建或刷新 connector。
4. 让 Thinking 读取项目、复核、写回 Codex。
```

如果需要终端方式：

```bash
cgn mcp connect --yes --open
cgn mcp trace
```

## 4. 安装到 Codex

第一次在项目里安装：

```bash
npx --yes --package github:rp10000/chatgpt-native-bridge -- cgn setup --mcp
```

安装后如果 Codex 提示需要重启，就重启 Codex。

在 Codex 里触发：

```text
$chatgpt-native-bridge
```

或者：

```text
请使用 chatgpt-native-bridge 处理这个任务。
```

## 5. 手动备用路径

如果桌面客户端和 MCP 都不可用：

```bash
cgn handoff --task "复核这个项目"
cgn done
```

Codex 最后读取：

```text
.chatgpt-native/inbox/{id}/reply.md
.chatgpt-native/inbox/{id}/CODEX_READ_THIS.md
```
