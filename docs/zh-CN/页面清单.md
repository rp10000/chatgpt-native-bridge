# 当前系统页面与内容清单

整理时间：2026-06-17  
当前版本：1.0.0  
整理范围：桌面客户端、本地网页 fallback、MCP/Actions HTTP 入口、ChatGPT 连接器页面、自动生成的 handoff/inbox 文档、README 介绍页。

## 1. 桌面客户端主页面

入口：

```bash
cgn start
cgn desktop
cgn client
```

源码入口：

```text
desktop/main.js
desktop/renderer/index.html
desktop/renderer/app.js
desktop/renderer/styles.css
```

定位：普通用户的主入口。用户不需要记 MCP、tunnel、inbox 路径等细节。

页面内容：

| 区域 | 页面显示 | 作用 |
| --- | --- | --- |
| 顶部 | 头像、`ChatGPT Native Bridge`、`本地工作区` | 显示产品和当前工作区 |
| 项目列表 | 项目名、项目路径、状态灯、连接状态 | 选择当前项目，并区分不同项目的 MCP 连接 |
| 状态卡片 | `连接`、`命令`、`结果`、`Codex` | 快速看当前是否连接、是否有命令、是否有回复、Codex 是否可继续 |
| 主流程 | `连接 ChatGPT`、`开始处理`、`查看结果`、`交给 Codex` | 完成 MCP 到 Codex 的闭环 |
| 工作区时间线 | 当前任务、ChatGPT 调用记录、命令输出、文件变更、写回 Codex | 让用户看清 ChatGPT 做了什么 |
| Pro 辅助规划 | 给 Pro 的任务、打包按钮、等待回复、手动导入 | GPT-5.5 Pro 不能直接读本地项目，只能用打包上下文辅助规划 |
| 诊断 | 日志、连接地址、错误信息、路径等 | 默认折叠，只在排查问题时看 |

主按钮：

| 按钮 | 当前作用 |
| --- | --- |
| `连接 ChatGPT` | 如果当前项目已有可用连接，则复用地址并打开 ChatGPT 连接器设置页；如果没有可用连接，才创建新连接 |
| `开始处理` | 复制给 ChatGPT 的短句：让 ChatGPT 使用 `chatgpt-native-bridge` 处理当前项目并写回 Codex |
| `查看结果` | 刷新命令输出、文件变更和最新写回 |
| `交给 Codex` | 复制给 Codex 的短句：`读取最新 Bridge 回复，检查变更摘要，然后继续执行、测试和总结。` |

项目状态灯：

| 状态 | 含义 |
| --- | --- |
| 灰色 | 未连接 |
| 蓝色 | 当前项目连接地址可用 |
| 黄色 | ChatGPT 已访问，但还没有真正调用工具 |
| 绿色 | 已有回复写回 Codex |
| 红色 | 连接失效或项目不匹配 |

当前没有独立的 `打开 ChatGPT` 按钮。打开 ChatGPT 的行为合并在 `连接 ChatGPT` 里。

## 2. 本地网页 fallback 页面

入口：

```bash
cgn app
```

默认地址：

```text
http://127.0.0.1:47833
```

源码入口：

```text
src/app-server.js
```

定位：桌面客户端不可用时的备用网页 GUI。

页面内容：

| 区域 | 页面显示 | 作用 |
| --- | --- | --- |
| 顶部 | `chatgpt-native-bridge`、`Pro clipboard relay for Codex` | 说明这是 Pro 剪贴板接力页面 |
| 状态卡片 | `Codex`、`Relay`、`MCP`、`Inbox` | 显示本地检查、剪贴板接力、MCP 调用和 inbox 状态 |
| Pro 辅助规划 | 任务输入框、`Copy Pro Prompt`、`Open ChatGPT`、`Start Clipboard Watch` | 生成 Pro 上下文包，复制到剪贴板，并可监听 Pro 回复 |
| Manual Fallback | 回复粘贴框、`Send to Codex` | 剪贴板监听失败时手动导入 |
| Log | 日志输出 | 显示操作结果和错误 |

本地网页 API：

| 路径 | 方法 | 作用 |
| --- | --- | --- |
| `/` | GET | 打开 fallback GUI |
| `/assets/avatar.png` | GET | 读取头像 |
| `/health` | GET | 返回 GUI 健康状态和版本 |
| `/api/status` | GET | 返回 doctor、handoff、MCP trace、relay 状态 |
| `/api/pro-pack` | POST | 创建 Pro 上下文包并复制 prompt |
| `/api/clipboard-watch/start` | POST | 开始监听当前 relay id 的剪贴板回复 |
| `/api/clipboard/import` | POST | 手动导入 Pro 回复 |
| `/api/open-chatgpt` | POST | 打开 ChatGPT |

## 3. MCP / Actions HTTP 入口

入口命令：

```bash
cgn mcp serve
```

默认本地地址：

```text
http://127.0.0.1:47832
```

源码入口：

```text
src/mcp-server.js
src/mcp-tools.js
src/mcp-web.js
```

面向 ChatGPT 的入口：

| 路径 | 类型 | 作用 |
| --- | --- | --- |
| `/health` | JSON | 返回 bridge 名称、版本、项目路径、项目名、项目 fingerprint |
| `/mcp` | MCP | ChatGPT Thinking / Developer Mode 调用的 MCP 服务入口 |
| `/action/openapi.json` | JSON | GPT Actions fallback 的 OpenAPI 描述 |
| `/action/review-current-project` | HTTP action | 复核当前项目 |
| `/action/read-repo-file` | HTTP action | 受限读取项目文件 |
| `/action/read-git-diff` | HTTP action | 读取当前 git diff |
| `/action/write-to-codex` | HTTP action | 写回 Codex inbox |

MCP 工具能力：

| 工具 | 作用 |
| --- | --- |
| `bridge_status` | 返回当前项目、git 状态、handoff/reply 状态 |
| `create_handoff` | 生成 handoff run |
| `list_handoff_files` | 列出 handoff 文件 |
| `read_handoff_file` | 读取 handoff 文件 |
| `read_repo_file` | 受限读取项目文件，禁止 `.env`、密钥、`.git`、路径穿越 |
| `read_git_diff` | 读取当前 diff 摘要或 patch |
| `submit_reply_to_codex` | 把 ChatGPT 回复写入 `.chatgpt-native/inbox` |
| `review_current_project` | 读取项目状态并生成复核材料 |
| `write_to_codex` | 将最终建议写回 Codex |

这些入口不是给普通用户手动浏览的页面，主要给 ChatGPT 工具调用。

## 4. ChatGPT 连接器页面

外部页面：

```text
https://chatgpt.com/#settings/Connectors
```

定位：用户在 ChatGPT 里创建或刷新 `chatgpt-native-bridge` 工具。

用户会看到或填写的内容：

| 字段 | 建议内容 |
| --- | --- |
| 名称 | `chatgpt-native-bridge` |
| 描述 | 本地 Codex bridge，用于读取项目上下文并写回 Codex |
| 连接方式 | Server URL |
| Server URL | 客户端复制出来的 `https://.../mcp` 地址 |
| 身份验证 | 无认证 / No authentication |

说明：

- Quick Tunnel 地址不是永久地址。
- 同一项目的 tunnel 仍然可用时，客户端会复用旧地址。
- 如果地址失效，客户端才会生成新地址。
- GPT-5.5 Pro 不能直接调用这个工具；Thinking / Developer Mode 才是读取本地项目的主路径。

## 5. 自动生成的 handoff 文档页面

入口命令：

```bash
cgn handoff
cgn ask
```

生成目录：

```text
.chatgpt-native/outbox/{id}/
```

生成文件：

| 文件 | 内容 |
| --- | --- |
| `START_HERE.md` | 给用户看的第一步说明：打开 ChatGPT、粘贴 prompt、上传文件、复制回复、再回 Codex |
| `01_PASTE_TO_CHATGPT.md` | 要粘贴给 ChatGPT 的主 prompt |
| `02_UPLOAD_THESE_FILES.md` | 建议上传的上下文文件列表和安全提醒 |
| `03_AFTER_CHATGPT_REPLY.md` | ChatGPT 回复后怎么导入、怎么让 Codex 继续 |
| `manifest.json` | 本次 handoff 的元数据 |

定位：没有 MCP 或桌面客户端不可用时的兜底路径。

## 6. 自动生成的 Codex inbox 文档页面

生成目录：

```text
.chatgpt-native/inbox/{id}/
```

生成文件：

| 文件 | 内容 |
| --- | --- |
| `reply.md` | ChatGPT 或 Pro 的原始回复 |
| `CODEX_READ_THIS.md` | 给 Codex 的读取说明：读取回复、判断采纳/拒绝/延后、继续实现、运行测试、总结结果 |

Codex 用户最终只需要说：

```text
读取最新 Bridge 回复，检查变更摘要，然后继续执行、测试和总结。
```

## 7. README / 介绍页

文件：

```text
README.md
README.zh-CN.md
```

定位：GitHub 和 npm 上给新用户看的介绍页。

主要内容：

| 区域 | 内容 |
| --- | --- |
| 项目标题 | `chatgpt-native-bridge` |
| 项目定位 | 给 Codex 使用的本地桌面桥接客户端 |
| 主路径 | 连接 ChatGPT 工具 -> ChatGPT Thinking 读取项目 -> 写回 Codex |
| 辅助路径 | Pro 辅助规划 -> 客户端复制上下文 -> 写回 Codex |
| 备用路径 | Markdown handoff |
| 第一次设置 | `npx --yes github:rp10000/chatgpt-native-bridge setup --mcp` |
| 常用命令 | `cgn start`、`cgn app`、`cgn mcp trace`、`cgn handoff`、`cgn done` |
| 安全边界 | 不用 API key、不抓 ChatGPT 网页、不走隐藏接口；MCP 工作区可运行命令和改文件，但不自动 commit/push |
| 桌面端开发 | `npm install`、`npm run desktop:dev`、`npm run desktop:pack` |
| 当前状态 | `v1.0.0`：MCP 工作区是主路径，加入项目白名单、搜索、目录、命令历史、桌面工作区时间线和 ChatGPT 网页端卡片 |
