# 当前系统页面与内容清单

整理时间：2026-06-18
当前目标：v1.2 Web-first MCP 极简客户端
主线：ChatGPT 网页端直接操作当前项目，桌面客户端只负责连接、状态和安全记录。

## 1. 桌面客户端主页面

入口：

```bash
cgn start
cgn desktop
cgn client
```

源码：

```text
desktop/main.js
desktop/renderer/index.html
desktop/renderer/app.js
desktop/renderer/styles.css
```

页面内容：

| 区域 | 显示内容 | 作用 |
| --- | --- | --- |
| 顶部 | 当前项目名、项目路径 | 让用户确认 ChatGPT 正在操作哪个项目 |
| 中间 | 大状态灯 | 显示连接和调用状态 |
| 底部 | `选择项目`、`连接 ChatGPT`、`生成交接报告` | 三个主操作 |
| 过程 | 调用记录、命令摘要、文件变更、报告状态 | 默认收起或简要展示 |
| 备用方式 | Pro 辅助规划、Markdown handoff | MCP 不可用时使用 |
| 诊断 | MCP URL、trace、命令历史、错误详情 | 排查问题时查看 |

状态灯：

| 颜色 | 含义 |
| --- | --- |
| 灰色 | 未连接 |
| 蓝色 | 已连接 |
| 黄色 | ChatGPT 已访问 |
| 绿色 | ChatGPT 正在操作当前项目 |
| 紫色 | 已生成交接报告 |
| 红色 | 项目不匹配或连接失效 |

## 2. ChatGPT 网页端

定位：主工作区。

ChatGPT 通过 MCP 可以调用：

```text
open_workspace
read
write
edit
bash
show_changes
create_handoff_report
```

网页端会显示卡片：

| 卡片 | 内容 |
| --- | --- |
| 项目卡片 | 当前项目和打开状态 |
| 命令卡片 | 命令、退出码、输出摘要 |
| 文件卡片 | 写入或编辑的文件摘要 |
| 变更卡片 | git status、diff 摘要、最近操作 |
| 报告卡片 | 交接报告 id 和路径 |

ChatGPT 默认只能操作客户端当前选中的项目。请求其它路径会被拒绝。

## 3. MCP / Actions HTTP 入口

入口命令：

```bash
cgn mcp serve
cgn mcp connect --yes --open
```

主要 HTTP 路径：

| 路径 | 类型 | 作用 |
| --- | --- | --- |
| `/health` | JSON | 返回 bridge 版本、当前项目、项目 fingerprint |
| `/mcp` | MCP | ChatGPT Developer Mode / Apps 连接入口 |
| `/action/openapi.json` | JSON | GPT Actions 备用 schema |
| `/action/write-to-codex` | HTTP action | 兼容入口，实际生成交接报告 |

## 4. 交接报告

ChatGPT 完成网页端操作后，应调用：

```text
create_handoff_report
```

生成：

```text
.chatgpt-native/reports/{id}/HANDOFF_REPORT.md
.chatgpt-native/inbox/{id}/CODEX_READ_THIS.md
```

Codex 最后读取报告，复核真实 diff，运行测试，再决定是否 commit / push。

## 5. 本地网页 fallback

入口：

```bash
cgn app
```

默认地址：

```text
http://127.0.0.1:47833
```

定位：桌面客户端不可用时的备用网页 GUI。它主要保留 Pro 剪贴板接力和手动导入能力，不是 v1.2 主流程。

## 6. Markdown 备用方式

入口：

```bash
cgn handoff
cgn done
```

生成目录：

```text
.chatgpt-native/outbox/{id}/
.chatgpt-native/inbox/{id}/
```

定位：没有 MCP 或桌面客户端不可用时的低风险兜底路径。

## 7. 当前没有的页面

| 页面 | 状态 |
| --- | --- |
| 登录/注册页 | 没有 |
| 浏览器插件页面 | 没有 |
| 自动控制 ChatGPT 网页的页面 | 没有 |
| 自动控制 Codex 输入框的页面 | 没有 |
| 自动 commit/push 页面 | 没有 |
| 全局文件系统管理页 | 没有 |

这些能力不做，是为了保持边界清楚：ChatGPT 只操作当前连接项目，Codex 做最终复核、测试、提交和 push。
